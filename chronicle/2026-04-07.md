## [Research] — Messaging Bot Landscape Survey

**What:** Broad product-decision survey of the full messaging bot + local AI landscape as of April 2026. Covered 7 areas: Telegram/LLM bots, other platforms (WhatsApp/Signal/Discord/iMessage), Claude API integrations, voice capture, security patterns, open source projects, and MCP as bridge.
**Why:** Needed comprehensive grounding before committing to PS Bot's interface layer — what to build vs. adopt vs. borrow from.
**Means:** The landscape crystallized sharply in March 2026 with Anthropic shipping Claude Code Channels (official Telegram/Discord/iMessage MCP plugin). This is the fastest path for basic Digital Core access. For PS Bot's full ambitions (voice, confidence-modulated prosody, lossless memory, TOTP-gated bash, daemon mode), community projects like claudeclaw and linuz90's personal assistant bot provide closest architectural patterns. OpenClaw (210k stars) is the dominant multi-platform alternative. Security finding: TOTP gating must be enforced pre-Claude in the bot handler layer, not as a Claude instruction. Custom MCP channels can extend Channels to Signal using the fully-documented reference spec.
