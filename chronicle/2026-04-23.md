**What:** Wisdom sent two reframes mid-session:

1. *"The permission-is-consciousness thing — let's just drop that for now. 2. *"We should have it be pretty clear that the status of this project is still a work in progress and especially the security — we're working heavily on it right now but it's not there yet. The claim is that you can just turn any Claude Code session into a persistent entity operating from your own Digital Core, using all the rules and skills you've built for Claude Code, now operating autonomously. And we've built and are building strong protections against reckless and dangerous reading and writing."*

**Why:** The permission-as-consciousness thesis had been the headline framing of the concept paper written earlier today. Wisdom's call is that the simpler, more concrete claim — *"turn any Claude Code session into a persistent entity operating from your own Digital Core, using all the rules and skills you've already built, with strong security protections in active development"* — is what the repo should lead with. The philosophical thesis is preserved (in the concept paper and Companion archive) but not foregrounded. And the work-in-progress status — especially on security — should be visible at the top, not buried.

**Means:**

- README.md opening rewritten to lead with the Digital Core angle and the autonomous-operation unlock.
- A visible "Status: Work in Progress" banner added at the top of the README (after the subtitle, before the one-paragraph claim).
- "How identity persists — the .md stack" section explains the concrete mechanism.
- "What's planned — dreaming, and what comes with it" section ties the three-drift-types research + dream-skill-research directories into roadmap framing.
- The Permissions section renamed to "Permissions — *security posture is actively being built*" and rewritten to name explicitly that the current posture is not yet adequate for unattended autonomous operation.

- HISTORY.md includes an explicit note documenting this framing pivot for future coherence.
- Engagement sections lead with a status-caveat sentence so visitors understand the work-in-progress nature before reading the audience framings.
