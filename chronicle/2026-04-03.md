## [Research] — Confidence-Prosody Psycholinguistics Deep Dive

**What:** Ran a full academic literature sweep on confidence/uncertainty expression in speech, affective TTS architectures, turn-taking models, and voice agent implementations. Drafted a complete `confidence_to_prosody()` mapping function with SSML rendering.
**Why:** PS Bot's confidence architecture needs a principled prosody layer. The TTS backend and turn-management behavior both need grounding before implementation.
**Means:** 20+ sources synthesized (Goupil 2021 Nature Comms, Pon-Barry 2008, Kirkland 2023, Singh 2025, JAL-Turn 2026, GST Wang 2018, CosyVoice 3.0, LiveKit, AssemblyAI). Mapped to concrete Python dataclass + rendering functions.
**Files:** `research/confidence-prosody-mapping.md` (new)

Key discoveries:
1. Confidence and accuracy have **separable prosodic signatures**: loudness tracks accuracy (calibration), duration/intonation track subjective confidence — both fields belong in the confidence object
2. The cross-linguistic certainty signature: falling intonation + lower pitch SD + faster rate + louder onset. Mean F0 is NOT significant — folk assumption debunked
3. No existing TTS system implements a confidence-to-prosody mapping — genuinely novel
4. Risk: excessive filled pauses in AI voice backfire (perceived as malfunction). Use prosodic markers (rate, intonation) primarily; linguistic markers sparingly
5. Turn-taking architecture: Silero VAD + semantic EOT + CNN interruption classifier. Confidence modulates yield threshold, response gap, backchannel rate

### The Process — Worth Documenting
