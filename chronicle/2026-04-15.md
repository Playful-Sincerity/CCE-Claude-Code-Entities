## [Evening] — Research

**What:** Deep source-level analysis of KAIROS internals — read 12 source files across autoDream, consolidationPrompt, memdir, cronScheduler, cronTasks, BriefTool, bootstrap/state, main.tsx, prompts.ts, REPL.tsx, and ScheduleCronTool.
**Why:** KAIROS was confirmed as 4/5 of our novelty combination in the landscape synthesis — needed to understand exactly how Anthropic implemented the daily-log memory pattern, tick loop, and cron scheduling to inform what we adopt vs build.
**Means:** Key discoveries: (1) autoDream is deliberately disabled when kairosActive — KAIROS uses a permanent installed cron task for dreams, not autoDream. (2) Memory is strictly append-only daily logs during sessions; nightly dream distills to MEMORY.md. (3) The tick is a cron-injected `<tick>` prompt, not a WebSocket or persistent process — the model uses SleepTool to control its own wakeup frequency. (4) BriefTool (SendUserMessage) is the sole user-facing channel in KAIROS — all output flows through it. (5) Permanent cron tasks (catch-up, morning-checkin, dream) are written at install time via writeIfMissing(), never recreatable. This is a complete blueprint for PD's heartbeat + memory architecture.

**Files:** `research/2026-04-15-kairos-source-analysis.md`

## — [Discovery]

**What:** Second-pass KAIROS source analysis found three major subsystems the first pass underweighted: Channels (messaging platform integration), Proactive autonomy calibration (terminal focus awareness), and force-async subagents.
**Why:** First pass covered the core 17 files but skipped 67 of 84 KAIROS-referencing files. Wisdom asked whether we'd documented everything inspirational.
**Means:** (1) Channels (`KAIROS_CHANNELS`) is the messaging layer — any MCP server declaring `claude/channel` becomes a communication channel (Slack, Telegram, iMessage, SMS). Permission dialogs relay to channels so you can approve operations from your phone. This is exactly the messaging layer PD needs — no custom integration required. (2) The proactive section injects terminal focus awareness — entity calibrates autonomy based on whether the human is watching. (3) KAIROS force-asyncs ALL subagents to prevent blocking the heartbeat. Also documented what KAIROS is NOT: no entity creation primitive, no identity layer, no multi-entity coordination, no identity development, no graduated autonomy. The novelty gap remains real. Analysis and landscape both copied to `claude-code-main/research/` (symlinked to Autonomous Claude).
